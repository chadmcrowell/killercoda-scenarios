# OOMKilled - Memory Limit Disasters

An OOMKilled container is one of the most disruptive failures in Kubernetes because it happens instantly with no warning and leaves no logs from the killed process. In this lab you will learn to identify OOMKilled events, understand what is causing memory exhaustion, and set memory limits that match real application behavior.
