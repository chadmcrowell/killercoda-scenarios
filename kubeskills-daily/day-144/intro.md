# Kubernetes Audit Logging - Capturing Security Evidence

Welcome to the audit logging lab. A secret was deleted from the production namespace last night and nobody on the team admits to doing it. Unfortunately this cluster has no audit logging configured, so there is no way to determine who is responsible. Your job is to configure audit logging correctly so this situation never repeats, and then practice using audit logs to trace operations.
